## в—Ҳ TF-IDF RAG вҖ” How the Retrieval Works

Zophiel does not use a neural embedding model. It uses **TF-IDF** (Term Frequency вҖ” Inverse Document Frequency), a deterministic mathematical method:

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ                   TF-IDF EXPLAINED                        в”Ӯ
в”Ӯ                                                           в”Ӯ
в”Ӯ  TF(t, d)  =  how often term t appears in document d     в”Ӯ
в”Ӯ                                                           в”Ӯ
в”Ӯ  IDF(t)    =  log( N / df(t) )                           в”Ӯ
в”Ӯ               N   = total documents                       в”Ӯ
в”Ӯ               df  = documents containing term t           в”Ӯ
в”Ӯ                                                           в”Ӯ
в”Ӯ  Score(t,d) = TF Г— IDF                                   в”Ӯ
в”Ӯ                                                           в”Ӯ
в”Ӯ  High score = term is common in THIS doc but rare overall в”Ӯ
в”Ӯ             = the term is distinctive and meaningful      в”Ӯ
в”Ӯ                                                           в”Ӯ
в”Ӯ  Query вҶ’ score all docs вҶ’ rank вҶ’ return top K            в”Ӯ
```

**Why not embeddings?**  
Embeddings (like those in GPT) require a neural model that can "hallucinate" relevance. TF-IDF is mathematically exact вҖ” if the term is not in the corpus, it will not fabricate a match.
